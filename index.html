Introduction

Refer a Friend program attracts good source of deposits at Capital One 360.

Components

  • Landing Pages
  • RAF API
  • Integration with Open Account application
  • Promotion Codes maintenance
  • Fulfillment

Landing Pages

Any promotion requires a landing page which will be marketed to targeted customers or all customers.

RAF API

Refer a Friend API is currently developed to support tracking referrals and send the referral information to 360 backends (Profile and CST).

Integration with Open Account application

Open Account application, in this case OAO interacts with RAF API to get the referral.

Promotion Codes maintenance

A Batch job is available to maintain Promotion Codes. Each promotion requires a Promotion Code which will have associated bonus amount for both Referrer and Referree. Promotion codes are maintained in CST Database.

Fulfillment

RAF Promotions are being fulfilled by Profile. Non RAF promotions are being fulfilled by Fulfillminator.

Current State

Landing Pages - Architecture

Landing pages are currently being developed using Andromeda technology.

RAF API - Architecture

RAF API is developed using Chassis framework and is currently being deployed using Artemis.

Promotion Batch Job - Architecture

Promotions Batch Job is developed using Spring Batch framework which is currently deployed on AWS Lambda.

Fulfillment

Fulfillment is being done by Profile system as Bonus amount need to be deposited into one of the 360 accounts.

Future State

Landing Pages - Architecture

Landing pages will still be developed using Andromeda technology.

RAF API - Architecture

As RAF API interacts with CST and Profile Systems, it will be enhanced to have the referral details maintained with in its own Database.

Fulfillment

As there are no bonus amounts going forward, the points fulfillment will be maintained by 3rd Party.

Future State - New Client Onboarding

Landing Pages - Architecture

Landing pages will be developed using Andromeda technology.

UI for onboarding new LOBs/Products

As RAF API is going to have its own Database for tracking referral information, onboarding new LOB/Product will need to be maintained in the same Database. A new UI application will be developed for onboarding a new LOB/Product.
